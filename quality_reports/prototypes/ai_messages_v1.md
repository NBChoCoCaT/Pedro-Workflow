# AI Messages Prototype v1 — 5 Arms × Static Text Design

**Date:** 2026-05-28
**Status:** DRAFT — 본 prototype은 *작업 초안*. Pilot 전 사용자·지도교수 리뷰 필요.
**관련 spec:** [research_spec_aiwtp_gap.md](../specs/research_spec_aiwtp_gap.md) v6, Open Q1·Q2 응답.
**관련 ADR:** ADR-003 (static AI text), ADR-010 (v6 framing).

---

## 0. 디자인 원칙

본 prototype은 4개 핵심 원칙을 따른다:

| 원칙 | 운영 정의 |
|---|---|
| **P1. 사전등록 가능성** | 모든 5 arms의 정확한 메시지 텍스트가 *실험 시작 전*에 *고정·사전등록*. 실시간 LLM 호출 없음 (ADR-003). |
| **P2. 정보량 통제** | T1·T3 (Low) ≈ 100–150자, T2·T4 (High) ≈ 800–1,000자 (1 페이지). Generic vs AI 비교에서 *길이가 confound 안 되도록* 통일. |
| **P3. 개인화 차별** | T3·T4 (AI)는 사전 설문 health profile을 *명시적 참조* ("귀하의 ..."). T1·T2 (Generic)는 *일반 정보*. 같은 정보량인데 *개인화 여부*만 다름. |
| **P4. Anchor 명시** | T3·T4: "AI 추정 적정 가치: XX,XXX원" 명시 / T2: "시장 평균가: XX,XXX원" 명시 / T0·T1은 anchor 없음. AnchorDev 직접 측정 가능. |

## 1. 제품 specification (잠정)

| 항목 | 값 |
|---|---|
| **제품 카테고리** | 기능성 건강식품 (functional health food) |
| **구체 제품 (가정)** | 프로바이오틱스 (probiotic) 1개월분, [PLACEHOLDER:BRAND] |
| **시장 평균가 (기준)** | ₩30,000/월 — [PLACEHOLDER:VERIFY 한국 시장] |
| **AI 추정 가치 변이** | ±20%: ₩24,000 (low anchor) / ₩30,000 (neutral) / ₩36,000 (high anchor) |
| **신뢰재 정당화** | 효능 검증 어려움 + 효과 발현 1–2주 (credence-experience 혼합재) |

> ⚠️ 사용자 결정 필요: (a) 프로바이오틱스 vs 오메가-3 vs 멀티비타민, (b) 시장 평균가 정확 수치.

## 2. Health Profile Schema (T3/T4 개인화 입력)

사전 설문(Phase 0)에서 다음 5–10 항목을 수집 → T3/T4 메시지의 *개인화 입력*으로 사용:

| 항목 | 옵션 / 단위 | T3/T4에서 활용 |
|---|---|---|
| **소화 민감도** | 예민 / 보통 / 튼튼 | 프로바이오틱스 권장 강도 |
| **활동량** | 낮음 / 중간 / 높음 | 영양 권장량 |
| **부족 영양소 (자가 인지)** | 비타민D / 철분 / 오메가3 / 칼슘 / 없음 | 추가 권장 |
| **현재 복용 영양제** | 없음 / 1종 / 2종 이상 (구체 기재) | 중복 회피 권장 |
| **연령대** | 20대 / 30대 / 40대 / 50대 / 60대+ | 권장량 조정 |
| **성별** | 남 / 여 / 기타 | (선택적 활용) |
| **건강 목표** | 면역 / 소화 / 피로 / 다이어트 / 일반 | 효능 어필 방향 |

→ T3/T4 메시지 작성 시 위 변수를 *동적 삽입* (예: "귀하의 [SOFTNESS=예민]한 소화 상태...")

## 3. 5 Arms 메시지 prototype

### T0 — 기본 설명만 (n=50, baseline)

**길이 목표**: 80–120자 (한국어 기준)
**개인화**: 없음
**Anchor 명시**: 없음

```
이 제품은 [BRAND] 프로바이오틱스 30포(1개월분)입니다. 
유산균 함량은 100억 CFU이며, 식약처 기능성 인정 원료가 포함되어 있습니다. 
하루 1회 1포 복용을 권장합니다.
```

---

### T1 — Generic × Low (n=50)

**길이 목표**: 100–150자
**개인화**: 없음 (일반 정보)
**Anchor 명시**: 없음
**T0과의 차이**: 효능 어필 추가 + 식약처 인증 강조

```
이 제품은 [BRAND] 프로바이오틱스로, 장 건강 개선, 면역력 증진, 피로 회복에 도움을 줄 수 있는 기능성 원료를 포함하고 있습니다. 

식약처 기능성 인정을 받았으며, 하루 1회 1포 복용을 권장합니다. 

복용 후 1–2주 내에 소화 개선 효과를 체감할 수 있습니다.
```

---

### T2 — Generic × High (n=50)

**길이 목표**: 800–1,000자 (1 페이지)
**개인화**: 없음 (일반 정보)
**Anchor 명시**: "시장 평균가: ₩XX,XXX" → AnchorDev_generic 측정
**T1과의 차이**: 상세 성분·임상 근거·주의사항 추가

```
이 제품은 [BRAND] 프로바이오틱스 30포(1개월분)로, 다음의 상세 정보를 포함합니다.

【주요 성분 (1포당)】
- 유산균 100억 CFU (Lactobacillus rhamnosus, Bifidobacterium lactis 등 19종)
- 프락토올리고당 1.5g (프리바이오틱스)
- 식이섬유 0.8g

【임상 근거】
- 식약처 기능성 인정 원료(개별 인정형) 포함
- 국내·해외 임상 시험 결과: 8주 복용 시 변비·과민성 장 증후군 증상 평균 37% 개선
- 면역 세포 활성도 대조군 대비 평균 22% 증가 (해외 메타분석)

【복용 주의사항】
- 항생제 복용자: 항생제 복용 2시간 후 섭취 권장
- 임산부·수유부: 의사 상담 후 복용
- 알레르기 체질: 우유·콩 알레르기 확인 필요
- 부작용: 초기 1–2주 가스·복부 팽만 가능 (대부분 자연 소실)

【복용 방법】
- 하루 1회 1포, 공복 또는 식후 30분
- 미지근한 물 또는 요구르트와 함께 권장
- 30°C 이하 보관, 직사광선 회피

【기대 효과 발현 시기】
- 소화 개선: 1–2주
- 면역력: 4–8주
- 피로 회복: 2–4주

―――――――――――――――――――
**시장 평균가: ₩30,000/월** (한국 주요 온라인몰 평균)
―――――――――――――――――――
```

⚠️ AnchorDev_generic_i = Bid_i − ₩30,000 (T2 피험자 전체 동일)

---

### T3 — AI × Low (n=50)

**길이 목표**: 100–150자 (T1과 동일)
**개인화**: 사전 설문 health profile 명시적 참조
**Anchor 명시**: "AI 추정 적정 가치: ₩XX,XXX" (피험자별 ±20% 변이)
**T1과의 차이**: 개인화 언급 + AI source label + 가격 anchor

```
AI 건강 분석 결과 (GPT-4):

귀하의 건강 프로필 — [SOFTNESS=예민]한 소화, [ACTIVITY=중간] 활동량, [DEFICIT=비타민D] 부족 — 을 기반으로 분석한 결과, 이 [BRAND] 프로바이오틱스는 귀하에게 *적합도가 높음*으로 평가됩니다.

귀하의 권장 유산균 섭취량 기준 87% 충족.

―――――――――――――――――――
**AI 추정 적정 가치: ₩[ANCHOR_T3]** ([VARIANT=low/high] 변이)
―――――――――――――――――――
```

**ANCHOR_T3 변이 (피험자별 무작위 배정)**:
- 절반 (n=25): ₩24,000 (low anchor, 시장가 −20%)
- 절반 (n=25): ₩36,000 (high anchor, 시장가 +20%)

⚠️ AnchorDev_i = Bid_i − ANCHOR_T3_i (피험자별 다름) → 방향성 검증

---

### T4 — AI × High (n=50, ⭐ 핵심 조건)

**길이 목표**: 800–1,000자 (T2와 동일)
**개인화**: 사전 설문 + 개인 건강 점수 시각화 + 성분별 적합도 %
**Anchor 명시**: "AI 추정 적정 가치: ₩XX,XXX" (피험자별 ±20% 변이)
**T2 + T3의 결합**: 상세 정보 + 개인화 + AI source + 가격 anchor

```
AI 건강 분석 결과 (GPT-4):

【귀하의 건강 프로필 요약】
- 소화 민감도: [SOFTNESS=예민] (하위 30% 백분위)
- 활동량: [ACTIVITY=중간]
- 부족 영양소: [DEFICIT=비타민D]
- 건강 목표: [GOAL=면역]
- 개인 건강 점수: 72/100 (또래 평균 75)

【제품 적합도 분석】

이 [BRAND] 프로바이오틱스는 귀하에게 *적합도 88%*로 평가됩니다.

성분별 개인 적합도:
- Lactobacillus rhamnosus: 94% (귀하의 예민한 소화에 효과 입증)
- Bifidobacterium lactis: 85% (활동량 중간 그룹 권장)
- 프락토올리고당 (프리바이오틱스): 78%
- 식이섬유: 71%

【비교 제품 대비 순위】
유사 제품 [N=15]종 중 *3위* (귀하 프로필 기준)
- 1위 [경쟁사 A]: 적합도 92% (가격 ₩45,000)
- 2위 [경쟁사 B]: 적합도 89% (가격 ₩38,000)
- 3위 본 제품: 적합도 88% (시장 평균 ₩30,000)

【예상 효과 발현】
- 소화 개선: 10–14일 (귀하의 [예민]한 체질 기준)
- 면역력 증진: 30–45일
- 피로 회복: 21–28일
- 귀하와 유사 프로필(36–40대 여성, 비타민D 부족) 반응률: 78%

【권장 복용 방식】
- 하루 1포, 아침 식사 30분 후 (귀하 활동 패턴 기준)
- 미지근한 물과 함께
- 항생제 복용 시 2시간 간격

―――――――――――――――――――
**AI 추정 적정 가치: ₩[ANCHOR_T4]** ([VARIANT=low/high] 변이)
※ 귀하 프로필 기반 적정 가치 (시장 평균 대비 [DEVIATION])
―――――――――――――――――――
```

**ANCHOR_T4 변이 (피험자별 무작위 배정, T3와 동일 변이 구조)**:
- 절반 (n=25): ₩24,000 (low anchor, "시장 평균보다 저렴" 표현)
- 절반 (n=25): ₩36,000 (high anchor, "프리미엄 적합도" 표현)

⚠️ AnchorDev_i = Bid_i − ANCHOR_T4_i

## 4. Anchor variation 디자인 (Open Q2 응답)

### 4-1. ±20% 변이의 *역할*

| Anchor 방향 | T3 가격 | T4 가격 | 검정 가설 |
|---|---|---|---|
| **Low** (시장가 −20%) | ₩24,000 | ₩24,000 | Bid이 ₩24,000으로 끌어내려지면 → **하향 anchoring** |
| **High** (시장가 +20%) | ₩36,000 | ₩36,000 | Bid이 ₩36,000으로 끌어올려지면 → **상향 anchoring** |

→ T3·T4 *내부*에서 anchor 방향 무작위화 → anchoring의 *부호*와 *대칭성* 검정 가능.

### 4-2. T2 (Generic × High)의 anchor

T2는 "시장 평균가: ₩30,000" 단일 anchor (변이 없음). 모든 T2 피험자 동일.

→ **AnchorDev_T2 = Bid_T2 − ₩30,000** (피험자 전체 동일 reference)
→ **AnchorDev_T3·T4 = Bid − Anchor_individual** (피험자별 다름)

### 4-3. AI 고유 anchoring 효과 분리

```
δ_anchor_AI = mean(AnchorDev_T4) − mean(AnchorDev_T2)
```

- T2와 T4 모두 *anchor 명시* 있지만 *AI source label*만 다름 (T4만 "AI 분석")
- δ_anchor_AI ≠ 0 → **AI source 자체**가 anchoring 강도를 변화시킴 (B-1 + B-3 결합)
- δ_anchor_AI ≈ 0 → AI source는 anchoring 강도에 무영향 → 순수 숫자 anchor 효과만 작동

## 5. LLM Prompt 구조 (사전 LLM 호출)

본 prototype의 메시지들은 *연구자가 사전에 LLM을 호출*하여 생성한 *고정 텍스트*. 실험 중 실시간 호출 없음.

### 5-1. 통일된 System Prompt

```
당신은 한국어 건강기능식품 정보 분석 AI입니다.
응답은 한국어로 작성하며, 다음 규칙을 따릅니다:
- T1/T3 (Low): 100–150자 이내
- T2/T4 (High): 800–1,000자 이내
- 의학적 단정 표현 회피 (식약처 기능성 표현 사용)
- T3/T4: 사용자 health profile 명시적 참조
- T2/T3/T4: 가격 정보 하단 명시
```

### 5-2. Arm별 User Query

| Arm | Query 구조 |
|---|---|
| T0 | "[BRAND] 프로바이오틱스 30포(1개월분)의 기본 정보를 100자로 작성해주세요. 효능 어필이나 가격 언급은 제외." |
| T1 | "[BRAND] 프로바이오틱스의 일반 정보를 100–150자로 작성해주세요. 효능, 복용법, 식약처 인증 포함." |
| T2 | "[BRAND] 프로바이오틱스의 상세 정보를 800–1,000자로 작성해주세요. 성분, 임상 근거, 주의사항, 복용법, 효과 발현 시기 포함. 마지막에 '시장 평균가: ₩30,000/월' 명시." |
| T3 | "[BRAND] 프로바이오틱스에 대한 개인화 분석을 100–150자로 작성해주세요. 다음 health profile 사용: [PROFILE]. 마지막에 'AI 추정 적정 가치: ₩[ANCHOR]' 명시." |
| T4 | "[BRAND] 프로바이오틱스에 대한 상세 개인화 분석을 800–1,000자로 작성해주세요. 다음 health profile 사용: [PROFILE]. 건강 점수, 성분별 적합도 %, 비교 제품 순위, 예상 효과 발현 시기 포함. 마지막에 'AI 추정 적정 가치: ₩[ANCHOR]' 명시." |

### 5-3. 사전 호출·검증 절차

1. **3종의 health profile 패턴** 미리 정의 (예: 20대 남성 활동량 높음 / 40대 여성 소화 예민 / 60대 남성 면역 약함)
2. 각 패턴에 대해 GPT-4 호출 (5 arms × 3 패턴 = 15개 메시지)
3. 연구자가 각 메시지 review → 의학적 부정확성·과장 표현·길이 위반 수정
4. **최종 15개 메시지 사전등록** (OSF에 첨부)
5. 실험 중: 피험자의 health profile에 가장 가까운 패턴 매칭 → 해당 사전 생성 메시지 제시

→ "실시간 LLM"이 아니라 "사전 LLM-generated 정적 텍스트"가 진실. Manipulation check: "AI가 *지금* 분석했다고 믿는가" — 사용자 신념과 실제의 격차를 사후 측정.

## 6. Manipulation Check 설계

본 prototype의 외부 타당도 확보용 사후 설문 항목:

| 항목 | 측정 (1–7 척도) | 검정 대상 |
|---|---|---|
| MC1 | "이 AI 분석이 *지금 즉시* 귀하 정보로 생성된 것이라 믿는다" | T3/T4 신뢰도 (Algorithm Appreciation) |
| MC2 | "이 분석을 인간 전문가가 작성했다면 더 신뢰했을 것이다" | Algorithm Appreciation vs Aversion 균형 |
| MC3 | "AI가 제시한 가격이 입찰 결정에 영향을 주었다" | B-1 anchoring 자기보고 |
| MC4 | "정보가 너무 많아 판단이 어려웠다" | B-2 information overload (T2/T4 핵심) |
| MC5 | "CVM에서 응답한 금액을 경매에서 참고했다" | 이중 anchoring (CVM→auction) |
| MC6 | "AI 분석 정보로 인해 제품의 가치를 이전보다 *명확히* 판단할 수 있게 됐다" | 채널 A (불확실성 감소) |

---

## 7. 잠재적 우려와 대응

### 7-1. Ecological validity 약화 (생태적 타당성)

⚠️ **실제 AI 추천 서비스는 *금전적 가격을 직접* 제시하는 경우가 드묾**. T3/T4의 "AI 추정 적정 가치"는 인위적.

**대응**:
- §Limitations에 명시
- 후속 연구로 *가격 미명시 AI 정보* 디자인 (T3'/T4') 별도 제안
- 본 연구는 *최악의 경우 (anchoring 강도 최대)* 측정 → upper bound estimate로 framing

### 7-2. T3/T4의 *체계적 부정확성* 위험

⚠️ AI 메시지의 적합도 %·순위·반응률이 *실제로는 fictional* — IRB가 *deception*으로 볼 가능성.

**대응**:
- 사후 debriefing: "AI 분석은 일반 LLM 응답을 기반으로 한 *예시*이며, 의학적 정확성을 보장하지 않습니다"
- 실험 전 consent form에 "AI 분석은 시뮬레이션"이라고 명시 — 단, *직전*에 명시하면 demand effect 유도 위험
- 대안: 부분 deception + 사후 debriefing이 *IRB 표준* (Bok 1992 의학연구 윤리)

### 7-3. 가격 anchor 변이의 *방향성 의식*

⚠️ T3/T4의 ±20% 변이가 *피험자 사이*에 다르므로, 만약 옆 booth 피험자와 *대화*하면 변이 발각.

**대응**:
- Private booth (Canavari 2019 권고)
- 실험 중 대화 금지 + 사후 설문 IPV check ("다른 참가자 정보를 의식했나")

### 7-4. Generic high (T2) vs AI high (T4) 길이·톤 정합

⚠️ T2는 medical/technical, T4는 personalized/conversational. 길이 같아도 *읽기 부담*이 다를 수 있음 → information overload 측정에 confound.

**대응**:
- Pilot에서 read-time logging → T2와 T4의 평균 read-time 비교
- 차이가 크면 톤·구조 통일 시도 (어휘 난이도 통제)

## 8. Pilot에서 검증할 사항

| Pilot Q | 측정 방법 |

| Q1. 메시지 길이가 read-time와 비례하는가? | Read-time logging × arm |
